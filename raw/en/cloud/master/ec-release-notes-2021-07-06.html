Billing Costs Analysis API, and moreedit

Access costs information for your organization through the API!

You can use the new Billing Costs Analysis API to fetch costs information for your organization. The following endpoints are avaialble:

  • GET /billing/costs/{organization-id}?from=start_date&to=end_date - fetch costs overview for the current organisation
  • GET /billing/costs/{organization-id}/deployments?from=start_date&to=end_date - fetch deployments costs for the current organisation
  • GET /billing/costs/{organization-id}/items?from=start_date&to=end_date - fetch itemized costs for the current organisation
  • GET /billing/costs/{organization-id}/deployments/{deployment-id}/items?from=start_date&to=end_date - fetch itemized costs by deployment

Depending on the role of the user consuming the API, end-user or administrator, the endpoints allow to fetch the costs of one’s own organization only or any organizations, respectively.

Enhancementsedit

Extend ConfigSettingsService to configure Enterprise Search setting kibana.external_url. Configure kibana.external_url for every Enterprise Search cluster.

Add password visibility to sign-up page. - Add show password functionality to the Elastic Cloud sign-up page.

New create deployment experience. Removed the ability to select different templates when creating a deployment and enhanced the overall deployment creation user experience.

Improved create deployment waiting experience. Simplified the user experience while a new deployment is being created.

Bug fixesedit

Fix move instance validation performance issue. Improved performance when validating moving instances.

Frozen autoscaling size display. Fixes the display of searchable storage sizing for Frozen tiers, when autoscaling is turned on.

Display frozen tier usage as MB. Fixed a unit conversion bug where we were displaying the number of bytes as the number of MB for frozen tier usage.
